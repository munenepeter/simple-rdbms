Minimal RDBMS in C

An educational implementation of a minimal relational database management system written in ISO C99. This project prioritizes transparency, simplicity, and fundamental building blocks over high performance or feature bloat.

C99 Educational Open Source

1 The Story

"this project was built as a part of an assessment of an open call for junior developers at pesal pal"

Pesapal is a leading payment services company focused on building innovative payment and business tools for African markets. The challenge was to design and implement a simple relational database management system (RDBMS) with support for:

  • Declaring tables with a few column data types
  • Full CRUD operations
  • Basic indexing (Primary and Unique keying)
  • Joining tables
  • Interactive SQL REPL mode

2 Why C?

"I have been learning and teaching myself C for the last year and therefore i saw this as the best project to practice what i have been learning, also i'd think for a RDBMS, it needs to be fast and very efficient with is storage, and to be honest there is no faster language i know, other that raw Assembly, that comes close to C when it comes to performance and efficiency."

"I know about it's "issues" with memory and it's reputation, but i'd say that's a skill issue on the developers side (that's why i did not use rust). plus am learning and think before i start with the borrow-checker introduces, i should first understand how to manage ownership directly myself"

3 Architecture Overview

The system follows a traditional UNIX-era database architecture with explicit stages for processing SQL commands.

SQL Lexer

Robust tokenization using stb_c_lexer.h. Converts raw SQL strings into a stream of meaningful tokens.

SQL Parser

Generates an Abstract Syntax Tree (AST) for CREATE TABLE, INSERT, SELECT, UPDATE, DELETE, and more.

Executor

Traverses the AST and calls the underlying storage engine to manipulate data and enforce constraints.

Storage Engine

Handles on-disk persistence, row serialization, catalog management, and automatic indexing for PRIMARY KEY and UNIQUE columns.

On-Disk Layout

data/
└── mydb/
    ├── db.meta          # Database header
    ├── catalog.meta     # Table and column metadata
    ├── tables/
    │   ├── users.tbl    # Row data for 'users'
    │   └── posts.tbl    # Row data for 'posts'
    └── indexes/
        ├── users_id.idx # Index for users.id
        └── users_email.idx # Index for users.email

4 Quick Start

Platform Note: This project has only been tested on Windows x64 systems. Pre-built executables are available in the Releases section.

Option 1: Using Pre-built Executable

# Download from Releases section
./db.exe mydb

Option 2: Build from Source

1. Build the System

mkdir build && cd build
cmake ..
cmake --build .

2. Initialize Database

./db.exe "" "CREATE DATABASE mydb;"

3. Launch REPL

./db.exe mydb

5 SQL Support

DDL: Create Table

CREATE TABLE users (
    id INT PRIMARY KEY,
    username TEXT UNIQUE NOT NULL,
    age INT
);

Supports PRIMARY KEY, UNIQUE, and NOT NULL constraints.

DML: Insert, Select, Update, Delete

INSERT INTO users VALUES (1, "alice", 25);
SELECT * FROM users WHERE id = 1;
SELECT name, id FROM users WHERE id = 10;
UPDATE users SET age = 26 WHERE id = 1;
DELETE FROM users WHERE id = 1;

Enforces constraints at insertion time. Selective columns, WHERE clauses, and full CRUD operations are supported.

Joins

SELECT * FROM users JOIN books ON users.id = books.user_id;

Supports INNER JOIN and OUTER JOIN operations.
